# 魔法学校インフレ育成シミュレーション 企画書

## 1. プロジェクト概要

| 項目 | 詳細 |
| --- | --- |
| **タイトル案** | アカデミア・マギカ：指数関数的成長の魔法学校 |
| **ジャンル** | テキストベース・アイドル（放置系）インフレ育成シミュレーション |
| **ターゲット** | 数値インフレ、育成の複合的な組み合わせ、戦略的なリセット（転生）要素を好むプレイヤー |
| **プラットフォーム** | Android (Jetpack Composeを想定) |

## 2. コアコンセプト

### 魔法学校経営と「総合魔力」のインフレ

プレイヤーは魔法学校の校長となり、限られたリソースの中で「学科」「施設」「生徒」の3要素をバランス良く育成します。育成の目的は、学校が生み出す**「総合魔力」**を指数関数的に増大させ、インフレの快感を追求することです。

### UIの制約と強調

- **UI要素は「テキスト」「数値」「プログレスバー」のみ**に限定。
- グラフィック要素を排除することで、数値の変化と育成要素の進捗に意識を集中させます。
- 特にインフレした数値（例: 1.25e+30）は、科学表記を用いてダイナミックに表現します。
- **【雰囲気とカラーリング】**
    - **テーマ:** ダーク・アカデミア（Dark Academia）と古の魔術書をイメージした、ミステリアスで厳かな雰囲気。
    - **背景色:** 深いインディゴまたはマットな漆黒（例: `#1A112A` / `#15151F`）。
    - **テキスト色:** 古びた羊皮紙のようなオフホワイトまたは薄い青灰色（例: `#E0E0FF`）。
    - **強調色/ハイライト:** 錬金術のシンボルのような、ミュートされたゴールドやブロンズ（例: `#C7A450`）。総合魔力の数値に適用。
    - **プログレスバー:** 育成の進行を「魔力の充填」として表現するため、深いエメラルドグリーンやアメジストパープルを使用し、静かに満ちていく感じを出す。

## 3. ゲームのコアサイクル (Core Loop)

ゲームは以下のサイクルを繰り返すことで進行し、インフレを加速させます。

1. **自動生成 (Idle Tick):** 生徒数と施設の効果に基づき、主要リソースである「マナ」や「ゴールド」が毎秒自動的に生成される。
2. **育成投資 (Upgrade):** 獲得したリソースを消費し、学科の研究、施設の改築、生徒の募集を行う。
3. **インフレ加速 (Inflation):** 育成の結果、各要素の効果が複合的に作用し、「総合魔力」が上昇する。
4. **周回リセット (Prestige):** 総合魔力が一定値に達したら「卒業式」や「時間の巻き戻し」といった名の**リセット（転生）を実行。育成要素は初期化されるが、代わりに永続的な乗算ボーナス（例: 賢者の石）**を獲得し、次周の成長速度を劇的に加速させる。

## 4. 主要な育成カテゴリと役割

育成要素は以下の3つの主要カテゴリに分類され、それぞれ異なる方法で総合値に貢献します。

### A. 学科・研究カテゴリ (複合的な乗算ボーナス)

リソースを消費してレベルを上げ、他の要素や総合魔力に対して**乗算**効果を付与する。レベル上限は「施設」カテゴリに依存。

| 学科 | 役割/ボーナス効果 | 進捗表現 |
| --- | --- | --- |
| **攻撃魔法 (🔥)** | **総合魔力の基本値**に直接的な加算または乗算ボーナス。インフレのベースを築く。 | Lv.X / プログレスバー |
| **植物学 (🌿)** | **リソース生産量**（マナ/ゴールド）に**乗算**ボーナスを付与し、育成の効率を高める。 | Lv.X / プログレスバー |
| **防衛魔法 (🛡️)** | **リソースの最大保有量**を引き上げる、または**募集コストをパーセンテージで削減**する。 | Lv.X / プログレスバー |
| **古代魔術 (📜)** | 周回リセット時に獲得できる**永続ボーナスの量**を増加させる。 | Lv.X / プログレスバー |

### B. 施設カテゴリ (上限値とシステム強化)

リソースを消費して施設を改築し、他のカテゴリの**最大値**や**システム的な効率**を強化する。

| 施設 | 役割/ボーナス効果 | 進捗表現 |
| --- | --- | --- |
| **大講堂** | **生徒の最大定員**（生徒数上限）を引き上げ、リソース生産の基礎を拡大する。 | 改築度 / 定員プログレスバー |
| **研究棟** | **学科レベルの最大上限**を引き上げ、乗算ボーナスをさらに高める道を開く。 | 建築レベル / 上限値プログレスバー |
| **次元図書館** | 学科の研究に必要なリソース消費量を**パーセンテージで削減**する。 | 蔵書数 / 削減率プログレスバー |

### C. 生徒・人材カテゴリ (基本生産力と特化加算)

自動リソース生産の基礎となる人数を確保し、特定の学科に特化した加算ボーナスを提供する。

| 要素 | 役割/ボーナス効果 | 進捗表現 |
| --- | --- | --- |
| **総生徒数** | **マナ/ゴールドの基本生産量**に直接貢献する。大講堂の定員に制限される。 | 総数 / 募集プログレスバー |
| **特待生** | 特定の学科（例: 植物学）の研究レベルに、**仮想的なレベルを加算**し、効果を増強する。 | 人数 / 特化度プログレスバー |

## 5. インフレの計算ロジック (複合乗算)

総合魔力の計算は、ゲーム終盤の「乗算」戦略が鍵となるよう設計します。

**【総合魔力 計算式】**

```
総合魔力 =
  (基本魔力_攻撃 × 生徒数ボーナス)
  × 施設乗数
  × 学科複合乗数
  × 永続ボーナス

```

- **基本魔力:** 攻撃魔法Lvに応じて線形増加
- **施設乗数:** 大講堂Lvや研究棟Lvに応じて乗算（例: **1.1 ^ (レベル)**）
- **学科複合乗数:** 複数の学科レベルの掛け合わせ（例: 植物学Lv × 防衛魔法Lv）
- **永続ボーナス:** 周回リセットで得た賢者の石の数などに応じた乗数
